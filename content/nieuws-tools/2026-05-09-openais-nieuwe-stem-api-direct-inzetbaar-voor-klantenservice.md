OpenAI heeft nieuwe voice-intelligence functies aan zijn API toegevoegd waarmee je spraakgestuurde toepassingen kunt bouwen. Voor MKB-ondernemers betekent dit dat je geautomatiseerde klantenservice via de telefoon kunt opzetten zonder dat je zelf een spraakmodel hoeft te trainen of te hosten.

## Wat er aan de hand is

OpenAI heeft de mogelijkheden van zijn API uitgebreid met voice-intelligence. Dat betekent dat ontwikkelaars nu spraakherkenning, stemherkenning en natuurlijke spraakgeneratie kunnen combineren in één API-call. Volgens TechCrunch zijn de nieuwe functies specifiek bedoeld voor klantenservicesystemen, maar OpenAI zegt dat ze ook toepasbaar zijn in onderwijs en op creator-platforms. De API werkt met dezelfde modellen die OpenAI ook voor ChatGPT gebruikt, maar dan via programmeerbare interfaces. De exacte prijzen per minuut of per verzoek zijn op moment van schrijven nog niet volledig bekendgemaakt, maar OpenAI rekent doorgaans per token of per API-call. Dit is een verschuiving ten opzichte van eerdere oplossingen, waarbij je aparte diensten moest combineren voor spraakherkenning, tekstverwerking en spraaksynthese.

## Wat dit betekent

Voor MKB-bedrijven wordt het een stuk eenvoudiger om een telefoonlijn te automatiseren. Waar je eerder een aparte speech-to-text dienst, een chatbot en een text-to-speech dienst aan elkaar moest knopen, kun je nu met één API-aanroep een compleet spraakgesprek opzetten. Dat verlaagt de technische drempel en de ontwikkeltijd. Voor een webshop met vijftig klantvragen per dag kan dit betekenen dat je eenvoudige vragen als ‘waar is mijn bestelling?’ of ‘wat zijn jullie openingstijden?’ volledig automatisch laat afhandelen. Voor een administratiekantoor kun je denken aan het inplannen van afspraken via de telefoon. De impact is het grootst voor bedrijven die nu nog een receptionist of medewerker hebben die vooral routinematige telefoontjes beantwoordt. Die tijd kun je vrijspelen voor complexere taken. Omdat OpenAI de modellen zelf host, hoef je geen eigen GPU’s of servers aan te schaffen. Je betaalt alleen voor wat je gebruikt.

## Hoe je dit kunt toepassen

**Als je een webshop runt met veel telefonische bestellingen of vragen**, overweeg om de voice-API te koppelen aan je bestelsysteem. Je zou een telefoonnummer kunnen instellen waar klanten hun bestelstatus kunnen navragen. De API herkent de klant op basis van stem of klantnummer, zoekt de order op in je database en leest de status voor. Een mogelijkheid is om dit eerst uit te proberen met een beperkt aantal vragen, zoals ‘waar is mijn pakket’ en ‘wat is de levertijd’. Je kunt de API zo instellen dat bij onduidelijke vragen het gesprek wordt doorgeschakeld naar een medewerker.

**Als je een team aanstuurt dat veel telefoontjes afhandelt**, kun je de voice-API inzetten als eerste filter. Je zou een menu kunnen bouwen waarin de beller zijn vraag inspreekt in plaats van door een toetsenkeuzemenu te navigeren. De API interpreteert de vraag en stuurt de beller naar de juiste afdeling of geeft direct antwoord. Een optie is om de API te trainen op de meest voorkomende vragen uit jouw sector, zodat hij die direct kan beantwoorden zonder tussenkomst van een medewerker. Dit werkt vooral goed voor vragen over openingstijden, adressen of veelgestelde productvragen.

**Als je in de zakelijke dienstverlening werkt**, bijvoorbeeld als administratiekantoor of verzekeringsadviseur, kun je de voice-API gebruiken voor het inplannen van afspraken. Je zou een systeem kunnen bouwen dat via de telefoon agenda-afspraken maakt. De API vraagt naar de gewenste datum en tijd, controleert de beschikbaarheid in je agenda en bevestigt de afspraak. Een mogelijkheid is om dit te combineren met een herinneringsfunctie: de API belt de klant een dag van tevoren om de afspraak te bevestigen. Dit bespaart heen-en-weer gemail en telefoontjes.

**Als je een onderwijsinstelling of trainingcentrum runt**, kun je de voice-API inzetten voor interactieve oefeningen. Studenten kunnen via de telefoon vragen stellen over de lesstof en krijgen direct antwoord. Je zou een systeem kunnen bouwen dat studenten helpt met het oefenen van talen of het stellen van vragen over studiemateriaal. De API kan meerdere talen aan, wat het geschikt maakt voor internationale klassen of cursussen.

**Als je een platform runt voor makers of creators**, kun je de voice-API gebruiken om audio-content te genereren of te bewerken. Denk aan het automatisch inspreken van blogposts of het genereren van voice-overs voor video’s. Je zou een tool kunnen bouwen waarin gebruikers tekst invoeren en de API die tekst uitspreekt in een natuurlijke stem. Dit kan handig zijn voor het maken van audioboeken, podcasts of instructievideo’s zonder dat je een voice-over hoeft in te huren.
