Mozilla heeft de afgelopen maanden 271 beveiligingslekken in Firefox opgespoord en gerepareerd, met hulp van een AI-model waar veel mensen bang voor zijn. Het model, Claude Mythos Preview van Anthropic, bleek juist een krachtig wapen tegen kwetsbaarheden in de browser. Voor ondernemers betekent dit dat AI niet alleen een risico is, maar ook een praktisch hulpmiddel om je eigen systemen veiliger te maken.

## Wat er aan de hand is
Mozilla maakte in een blogpost bekend dat het 271 beveiligingslekken in Firefox heeft gedicht, en dat het daarbij hulp kreeg van Claude Mythos Preview. Dit AI-model van Anthropic staat bekend als een van de krachtigste én meest omstreden modellen, omdat het volgens de maker zelf risico's met zich meebrengt. Mozilla gebruikte het model echter niet om code te schrijven, maar om bestaande code te analyseren op kwetsbaarheden. Het bedrijf zegt dat het zonder AI onmogelijk was geweest om zoveel lekken in zo'n korte tijd te vinden. De aanpak laat zien dat AI-modellen, zelfs de 'enge' varianten, een praktische en veilige toepassing kunnen hebben in softwareontwikkeling en beveiliging.

## Wat dit betekent
Voor ondernemers in het mkb is dit nieuws relevanter dan het lijkt. Beveiligingslekken in software zijn een van de grootste risico's voor bedrijven, of je nu een webshop runt, een administratiesysteem gebruikt of klantgegevens opslaat. De traditionele manier om lekken te vinden – handmatige code reviews en geautomatiseerde scanners – is tijdrovend en duur. Mozilla laat zien dat AI-modellen dit werk sneller en grondiger kunnen doen, zonder dat je een groot beveiligingsteam nodig hebt. Het betekent dat kleinere bedrijven straks toegang kunnen krijgen tot beveiligingsniveaus die voorheen alleen voor grote organisaties betaalbaar waren. Tegelijkertijd laat het zien dat de angst voor geavanceerde AI-modellen niet altijd terecht is: hetzelfde model dat als 'gevaarlijk' wordt bestempeld, kan juist worden ingezet om systemen te beschermen.

## Hoe je dit kunt toepassen
**Als je een webshop runt met maatwerk software...** Je kunt overwegen om een AI-model in te zetten om de code van je webshop te laten scannen op bekende kwetsbaarheden. Veel AI-diensten bieden tegenwoordig een 'security audit' functie aan, waarbij je code kunt uploaden en het model zwakke plekken aanwijst. Je zou kunnen beginnen met een klein deel van je codebase, bijvoorbeeld de betaalmodule, om te zien wat het oplevert.

**Als je een team aanstuurt dat zelf software ontwikkelt...** Een optie is om AI-modellen te integreren in je ontwikkelproces, bijvoorbeeld als onderdeel van de 'code review'. Laat het model elke nieuwe code controleren voordat deze in productie gaat. Mozilla's aanpak laat zien dat dit niet alleen veiliger is, maar ook tijd bespaart voor je ontwikkelaars, die zich dan kunnen richten op nieuwe functionaliteit in plaats van handmatig speuren naar fouten.

**Als je externe software gebruikt van leveranciers...** Je zou kunnen vragen of jouw leveranciers AI inzetten voor beveiliging. Als zij nog geen geautomatiseerde kwetsbaarheidsscans doen, is dat een signaal dat hun software mogelijk minder veilig is. Overweeg om dit onderdeel te maken van je inkoopvoorwaarden: vraag naar hun beveiligingsproces en of ze AI gebruiken om lekken te vinden.

**Als je zelf geen code schrijft maar wel een website beheert...** Een mogelijkheid is om een beveiligingsscan uit te voeren op je website met een AI-gestuurde tool. Er zijn diensten die je URL kunnen controleren op veelvoorkomende kwetsbaarheden zoals SQL-injecties of cross-site scripting. Je zou dit maandelijks kunnen doen, net zoals je een virusscan op je computer draait.

**Als je in de zorg of een andere gereguleerde sector werkt...** Overweeg om AI in te zetten voor het controleren van je software op naleving van privacyregels, zoals de AVG. Hetzelfde model dat lekken vindt, kan ook controleren of persoonsgegevens op de juiste manier worden verwerkt. Je zou kunnen beginnen met een kleine audit van een specifiek systeem, zoals je klantportaal.
