OpenAI’s nieuwste AI-modellen, waaronder Codex, zijn nu direct beschikbaar op Amazon Web Services (AWS). Dit betekent dat ondernemers met een bestaand AWS-contract deze modellen kunnen gebruiken zonder een aparte OpenAI-licentie of extra leverancier.

## Wat er aan de hand is

OpenAI heeft aangekondigd dat zijn frontier-modellen en Codex algemeen beschikbaar zijn op AWS. Codex is een AI-model dat code kan genereren en begrijpen. Voorheen moesten bedrijven die OpenAI wilden gebruiken, een apart account aanmaken bij OpenAI en aparte afspraken maken over beveiliging en facturatie. Nu kunnen ze via hun bestaande AWS-omgeving aan de slag. Volgens OpenAI kunnen klanten hierdoor sneller van evaluatie naar productie gaan, omdat ze gebruikmaken van de beveiligingscontroles en inkoopprocessen die ze al kennen. Dit is een verschuiving: grote AI-aanbieders zoals OpenAI worden steeds vaker onderdeel van bestaande cloudplatforms, in plaats van aparte diensten.

## Wat dit betekent

Voor MKB’ers met een AWS-contract verandert dit de manier waarop ze AI kunnen inzetten. Het grootste voordeel is dat je geen aparte leverancier hoeft te beheren. Je facturatie loopt via AWS, je beveiligingsinstellingen zijn hetzelfde, en je kunt AI-functionaliteit toevoegen zonder dat je een nieuw inkooptraject hoeft te doorlopen. Dit bespaart tijd en vermindert administratieve rompslomp. Vooral voor bedrijven die al werken met AWS voor hun hosting, data-opslag of applicaties, wordt de drempel om AI te gebruiken lager. De impact is het grootst voor bedrijven die software ontwikkelen of processen automatiseren, omdat Codex specifiek gericht is op het genereren van code. Maar ook voor andere toepassingen, zoals het analyseren van data of het genereren van tekst, zijn de OpenAI-modellen nu eenvoudiger toegankelijk.

## Hoe je dit kunt toepassen

**Als je een webshop runt en al gebruikmaakt van AWS voor hosting.** Je zou kunnen overwegen om OpenAI via AWS in te zetten voor het automatisch genereren van productbeschrijvingen. Omdat je al een AWS-omgeving hebt, kun je eenvoudig een API-verbinding maken zonder dat je een aparte OpenAI-licentie nodig hebt. Een mogelijkheid is om een script te schrijven dat nieuwe producten uit je database haalt en via de OpenAI API een beschrijving laat genereren, die je vervolgens direct in je webshop plaatst.

**Als je een team aanstuurt dat software ontwikkelt.** Codex kan je ontwikkelaars helpen om sneller code te schrijven. Overweeg om Codex via AWS beschikbaar te maken in de ontwikkelomgeving van je team. Omdat de facturatie via AWS loopt, kun je het gebruik eenvoudig monitoren en budgetteren. Een optie is om te beginnen met een pilot waarbij een paar ontwikkelaars Codex gebruiken voor het genereren van standaardcode, zoals API-koppelingen of data-validatie.

**Als je in de logistiek werkt en processen wilt automatiseren.** Je zou OpenAI kunnen gebruiken om automatisch rapporten te genereren op basis van data uit je AWS-database. Denk aan een wekelijks overzicht van voorraadniveaus of verzendstatistieken. Omdat je geen aparte leverancier hoeft in te schakelen, kun je dit snel zelf opzetten. Een mogelijkheid is om een AWS Lambda-functie te maken die elke week data ophaalt, via OpenAI een samenvatting laat schrijven en die per e-mail verstuurt.

**Als je klantenservice hebt en chatbots gebruikt.** Je kunt de OpenAI-modellen via AWS inzetten om een slimmere chatbot te bouwen die vragen van klanten beantwoordt. Omdat de beveiligingscontroles van AWS van toepassing zijn, kun je gevoelige klantdata veilig verwerken. Overweeg om te starten met een eenvoudige chatbot die veelgestelde vragen beantwoordt, voordat je uitbreidt naar complexere interacties.
