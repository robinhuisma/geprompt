De AI-coderingsassistent Claude Code van Anthropic heeft nu een stemmodus. In plaats van alleen maar te typen, kun je nu tegen de assistent praten om code te genereren, te bespreken of fouten te laten uitleggen. Dit verandert de interactie van een typdialoog in een gesprek, wat vooral handig kan zijn tijdens het brainstormen, snel prototypen of wanneer je handen vrij moeten blijven.

## Wat betekent dit voor een ontwikkelaar?
De kern is dat je nu een gesprek kunt voeren over je code. Je kunt hardop een probleem formuleren, zoals "waarom geeft deze functie een null reference error?", en Claude Code zal het uitleggen. Je kunt ook mondeling instructies geven, zoals "maak een React-component voor een login-formulier met validatie", en de AI schrijft de code terwijl je praat. Dit kan de snelheid van itereren verhogen, omdat het praten vaak sneller gaat dan typen, vooral voor complexe uitleg.

## De praktische voordelen op een rij
Een belangrijk voordeel is de mogelijkheid tot multitasking. Je kunt code reviewen of een architectuurdiagram bekijken terwijl je mondeling feedback geeft of aanpassingen doorgeeft. Het kan ook de drempel verlagen voor het vragen van hulp bij complexe problemen; het is intuïtiever om iets uit te leggen dan het in een chatvenster te moeten formuleren. Voor teams die pair programming doen of code reviews houden, opent dit de deur naar meer dynamische, gespreksgebaseerde sessies met de AI als actieve deelnemer.

## Hoe kun je dit vandaag toepassen?
De stemmodus is volgens het bericht nu aan het uitrollen in Claude Code. Een mogelijkheid is om de feature te proberen tijdens je volgende debug-sessie. In plaats van foutmeldingen te googlen of uitgebreid te typen, kun je de fout hardop voorlezen en om uitleg vragen. Een andere suggestie is om het te gebruiken voor snelle prototyping: spreek de functionaliteit in die je nodig hebt en laat de AI de eerste code-opzet maken, waarna je het gesprek kunt gebruiken om die opzet te verfijnen.
