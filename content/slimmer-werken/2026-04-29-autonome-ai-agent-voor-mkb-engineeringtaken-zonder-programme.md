Siemens heeft een AI-agent gelanceerd die zelfstandig engineeringtaken in de industriële automatisering uitvoert, van het schrijven van code tot het configureren van systemen. Voor MKB-bedrijven die worstelen met een tekort aan engineers, opent dit de mogelijkheid om complexe automatiseringstrajecten sneller en consistenter uit te voeren zonder dat er een volledige programmeur aan te pas hoeft te komen.

## Wat er aan de hand is

Siemens heeft op de Hannover Messe de Eigen Engineering Agent gepresenteerd, een AI-oplossing die engineeringtaken op het gebied van industriële automatisering zelfstandig kan uitvoeren. Volgens het bedrijf verschilt deze agent van bestaande AI-tools en copilots doordat hij niet alleen advies geeft, maar taken daadwerkelijk uitvoert binnen echte engineeringomgevingen. De agent kan automatiseringsprojecten doorgronden, code schrijven voor PLC’s (programmable logic controllers), HMI-visualisaties (human-machine interface) maken, systemen configureren en resultaten valideren. Dit herhaalt hij net zo lang tot vooraf vastgestelde prestatie-eisen zijn gehaald.

De agent is inmiddels getest bij meer dan honderd bedrijven in negentien landen, waaronder Andritz Metals uit Oostenrijk, Casmt uit China en Prism Systems uit de Verenigde Staten. Siemens stelt dat de agent taken twee tot vijf keer sneller uitvoert dan handmatige workflows, wat in specifieke toepassingen kan leiden tot een vijftig procent hogere engineeringefficiëntie. De oplossing is commercieel beschikbaar voor gebruikers van Siemens’ TIA Portal, dat wereldwijd meer dan zeshonderdduizend gebruikers telt, en maakt deel uit van het Siemens Xcelerator-portfolio.

## Wat dit betekent

Voor MKB-bedrijven in de industriële automatisering is dit een verschuiving van formaat. Tot nu toe betekende het automatiseren van een productielijn of machine dat je een gespecialiseerde engineer nodig had die uren of dagen bezig was met het schrijven van PLC-code, het ontwerpen van visualisaties en het testen van configuraties. De Eigen Engineering Agent kan deze taken zelfstandig uitvoeren, wat de drempel verlaagt voor bedrijven die geen vaste programmeur in dienst hebben.

Het tekort aan gekwalificeerde engineers is in de sector voelbaar. Vooral kleinere bedrijven hebben moeite om geschikt personeel te vinden of in te huren. Een AI-agent die een deel van het werk overneemt, kan betekenen dat je met minder specialisten toch complexere projecten kunt doen. De agent werkt binnen de bestaande Siemens-omgeving, dus bedrijven die al met TIA Portal werken, kunnen relatief eenvoudig instappen. Voor bedrijven die nog niet met Siemens werken, is de stap groter: je moet eerst het platform implementeren voordat de agent iets voor je kan doen.

## Hoe je dit kunt toepassen

**Als je een machinebouwer bent die PLC-programmering uitbesteedt.** Je hebt een klant die een nieuwe machine wil automatiseren, maar je hebt geen vaste programmeur in dienst. Je zou kunnen overwegen om de Eigen Engineering Agent in te zetten voor het genereren van de basis-PLC-code. Je geeft de specificaties aan, de agent schrijft de code en valideert deze. Jij controleert het eindresultaat. Dit kan de doorlooptijd van weken naar dagen verkorten.

**Als je een productiebedrijf runt met een kleine technische dienst.** Je team bestaat uit monteurs en een enkele engineer die overbelast raakt met storingen en aanpassingen. Een optie is om de agent te gebruiken voor routinematige configuratiewijzigingen, zoals het aanpassen van HMI-schermen of het toevoegen van sensoren aan een bestaande PLC. De engineer kan zich dan richten op complexe storingen, terwijl de agent de standaard taken afhandelt.

**Als je als zelfstandig engineer werkt en meer projecten wilt aannemen.** Je hebt een volle orderportefeuille, maar elke klus kost veel handwerk. Je zou de agent kunnen gebruiken als een digitale assistent die de eerste versie van de code schrijft. Jij beoordeelt, past aan en levert op. Dit kan betekenen dat je twee tot vijf keer sneller kunt werken, zoals Siemens aangeeft, en dus meer klanten kunt bedienen zonder extra personeel.

**Als je een machine wilt aanpassen maar geen tijd hebt om de handleiding te bestuderen.** Stel dat je een bestaande Siemens-besturing hebt en een extra functie wilt toevoegen, zoals een veiligheidscontrole of een nieuwe productmodus. De agent kan op basis van je beschrijving de benodigde configuratie genereren en testen. Je hoeft niet te weten hoe de code precies werkt, zolang het resultaat voldoet aan de eisen.

**Als je een team aanstuurt dat werkt met meerdere projecten tegelijk.** De agent kan consistentie brengen in de manier waarop code wordt geschreven en systemen worden geconfigureerd. Overweeg om de agent in te zetten als een standaardisatietool: elk project begint met een door de agent gegenereerde basis, waardoor fouten door verschillende werkwijzen afnemen. Dit is vooral waardevol als je teamleden wisselen of als je projecten overdraagt aan collega’s.
