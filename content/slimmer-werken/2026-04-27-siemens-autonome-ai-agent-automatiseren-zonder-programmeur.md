Siemens heeft een AI-agent gelanceerd die zelfstandig engineeringtaken uitvoert, zoals het schrijven van programmeercode en het configureren van machines. Dit betekent dat ook bedrijven zonder eigen programmeerafdeling complexe industriële processen kunnen automatiseren.

## Wat er aan de hand is

Siemens heeft op de Hannover Messe de Eigen Engineering Agent gepresenteerd. Het is een AI-oplossing die taken in industriële automatisering niet alleen adviseert, maar ook daadwerkelijk uitvoert. Volgens Siemens kan de agent automatiseringscode schrijven, systemen configureren en resultaten valideren, net zo lang tot de gestelde eisen zijn gehaald. De agent is getest bij meer dan honderd bedrijven in negentien landen, waaronder Andritz Metals, Casmt en Prism Systems. De oplossing is commercieel beschikbaar voor gebruikers van Siemens’ TIA Portal, een platform met wereldwijd meer dan zeshonderdduizend gebruikers. Siemens claimt dat de agent taken twee tot vijf keer sneller uitvoert dan handmatige workflows, met een mogelijke efficiëntieverbetering van vijftig procent in specifieke toepassingen. De aankondiging past in een bredere strategie: Siemens investeert een miljard euro in industriële AI en heeft meer dan 1.500 AI-specialisten in dienst.

## Wat dit betekent

Voor het mkb is dit relevant omdat het de drempel verlaagt voor procesautomatisering. Waar je voorheen een gespecialiseerde engineer nodig had voor het programmeren van PLC’s of het configureren van HMI-visualisaties, kan een AI-agent die taken nu grotendeels overnemen. Dat is vooral interessant voor bedrijven die kampen met een tekort aan gekwalificeerd personeel of die repeterende engineeringwerkzaamheden willen versnellen. De agent werkt binnen de bestaande Siemens-omgeving, wat betekent dat bedrijven die al met TIA Portal werken relatief eenvoudig kunnen starten. Voor bedrijven die nog niet met Siemens werken, is de stap groter: je hebt het platform nodig om de agent te kunnen gebruiken. De impact is het grootst in sectoren waar veel standaardisatie mogelijk is, zoals machinebouw, voedingsmiddelenindustrie en logistieke automatisering.

## Hoe je dit kunt toepassen

**Als je een productiebedrijf runt met veel herhalende engineeringtaken**, overweeg dan om de Eigen Engineering Agent in te zetten voor het standaardiseren van PLC-programmering. Je zou kunnen beginnen met een pilot voor één type machine of productielijn. De agent kan de code schrijven en testen, waardoor je eigen engineers vrijkomt voor complexere vraagstukken. Een mogelijkheid is om de agent te laten valideren of bestaande code voldoet aan de nieuwste veiligheidsnormen.

**Als je een klein ingenieursbureau bent dat automatiseringstrajecten begeleidt**, kun je de agent gebruiken om de doorlooptijd van projecten te verkorten. In plaats van dat een senior engineer elke configuratie handmatig doet, kan de agent een eerste ontwerp maken dat de engineer alleen nog hoeft te controleren. Je zou kunnen overwegen om de agent in te zetten voor het genereren van HMI-visualisaties op basis van een eenvoudige specificatie, wat normaal gesproken veel tijd kost.

**Als je als mkb’er nog niet met TIA Portal werkt**, is het zinvol om te onderzoeken of de investering in het platform zich terugverdient door de tijdsbesparing die de agent biedt. Je zou een kosten-batenanalyse kunnen maken voor een specifiek project, bijvoorbeeld het herprogrammeren van een bestaande productielijn. Een optie is om een demo aan te vragen bij Siemens om te zien hoe de agent presteert in jouw type toepassing.

**Als je verantwoordelijk bent voor kwaliteitsborging in een productieomgeving**, kun je de agent inzetten voor het valideren van configuraties. De agent kan controleren of alle parameters binnen de gestelde toleranties vallen en of de code voldoet aan de bedrijfsstandaarden. Dit vermindert het risico op fouten die achteraf duur zijn om te herstellen.
