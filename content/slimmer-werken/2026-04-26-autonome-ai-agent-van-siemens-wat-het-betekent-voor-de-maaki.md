Siemens heeft een AI-agent gelanceerd die zelfstandig engineeringtaken in de industriële automatisering kan uitvoeren, zoals het schrijven van code en configureren van systemen. Voor Nederlandse maakbedrijven betekent dit dat repetitief technisch werk mogelijk sneller en consistenter kan worden gedaan, zonder dat je zelf AI hoeft te ontwikkelen.

## Wat er aan de hand is

Siemens heeft op de Hannover Messe de Eigen Engineering Agent gepresenteerd. Dit is een AI-oplossing die taken in industriële automatisering niet alleen adviseert, maar ook daadwerkelijk uitvoert. Volgens Siemens kan de agent automatiseringscode schrijven, systemen configureren en resultaten valideren, net zo lang tot vooraf vastgestelde prestatie-eisen zijn gehaald. De agent is getest bij meer dan honderd bedrijven in negentien landen, waaronder Andritz Metals, Casmt en Prism Systems. De oplossing is commercieel beschikbaar voor gebruikers van Siemens’ TIA Portal, dat wereldwijd meer dan zeshonderdduizend gebruikers telt. Siemens claimt dat de agent taken twee tot vijf keer sneller uitvoert dan handmatige workflows, met een mogelijke efficiëntiewinst van vijftig procent in specifieke toepassingen. De introductie past in een bredere AI-strategie van Siemens, die een investering van een miljard euro in industriële AI omvat.

## Wat dit betekent

Voor de Nederlandse maakindustrie, waar veel MKB-bedrijven actief zijn in industriële automatisering, kan deze AI-agent een verschuiving betekenen in hoe engineeringwerk wordt gedaan. Het tekort aan gekwalificeerde engineers is in deze sector voelbaar, en repetitieve taken zoals PLC-programmering en HMI-visualisaties kosten nu veel tijd. Als de agent deze taken inderdaad sneller en consistenter kan uitvoeren, zoals Siemens beweert, dan kunnen bedrijven met een kleinere technische staf mogelijk meer projecten aan. De agent is gekoppeld aan het TIA Portal, een platform dat al breed wordt gebruikt in de Nederlandse industrie. Dat betekent dat de drempel om ermee te werken relatief laag is voor bedrijven die al Siemens-software gebruiken. Voor bedrijven die andere systemen gebruiken, is de agent vooralsnog niet direct toepasbaar.

## Hoe je dit kunt toepassen

**Als je een machinebouwer bent die PLC-programmering uitbesteedt.** Je zou kunnen onderzoeken of de Eigen Engineering Agent binnen jouw TIA Portal-omgeving taken zoals het schrijven van besturingscode kan overnemen. Een mogelijkheid is om een pilot te starten met een eenvoudig project, zoals het programmeren van een transportband, om te zien of de agent de doorlooptijd verkort. Overweeg om een engineer vrij te spelen voor complexere taken, terwijl de agent het repetitieve werk doet.

**Als je een technisch directeur bent in een MKB-fabriek.** Je zou kunnen kijken naar de inzet van de agent voor het configureren van apparaten of het maken van HMI-visualisaties. Een optie is om een bestaand project te hergebruiken als testcase, waarbij je de agent een configuratie laat maken en die vergelijkt met handmatig werk. Als de agent consistentie biedt, kun je mogelijk fouten in de productie verminderen.

**Als je een engineer bent die worstelt met werkdruk.** Je zou de agent kunnen gebruiken als assistent voor valideren van code of het doorlopen van prestatie-eisen. Overweeg om de agent in te zetten voor de laatste controleslag voordat je code naar de productieomgeving stuurt. Dit kan je tijd geven om je te focussen op ontwerpvraagstukken die meer creativiteit vragen.

**Als je een ondernemer bent die overweegt te investeren in automatisering.** Je zou kunnen nagaan of jouw huidige leveranciers of systeemintegratoren al met de agent werken. Een mogelijkheid is om hen te vragen of ze de agent kunnen inzetten in jouw project, zonder dat je zelf in de technologie hoeft te duiken. Dit kan een manier zijn om de efficiëntiewinst te benutten zonder eigen AI-ontwikkeling.
